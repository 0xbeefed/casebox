Instructions:

1. create config file in /casebox/httpsdocs/cores/$coreName/ (copy from another config and make necessary changes) 
2. create database for new core (cb_<core_name>, with charset utf8, collation utf8_general_ci) and copy contents from another core. 
3. Clear MySql log, users etc
4. update root password in MySql (UPDATE users SET `password` = MD5(concat('aero', 'new_password')) where `name` = 'root'; )
5. copy file structure from original core to new core if needed ( /casebox/data/files/ )
   this is needed if you clone a core with all the files
6. instruct solr to load a new core (/opt/solr/multicore/solr.xml). 
7. Restart jetty (service jetty restart, or net start jetty" on windows if you previously added it as a service)
8. Run manual script (from crons/) for solr core update (ex: php -f /sys/utils/solr_reindex_core.php <core_name>)
9. add apache alias for new host (/etc/httpd/conf.d/ssl_casebox.conf)
10. restart apache (service httpd restart)
